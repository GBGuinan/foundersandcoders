Project Based

The course emphasises project work building real-world applications: every week you’ll build slick websites, smart servers, and cool applications.

Self-Organising

Students take responsibility for organising themselves into agile product development teams. Learning is self-directed and driven by real world goals.

Free for Students

The course is free, so finances aren't a barrier to learning. Because it is free, we can be selective in accepting the most motivated applicants.

Curriculum

Pre-course Foundations of coding
  • HTML and CSS (Codecademy)
  • Javascript (Codecademy)
  • jQuery (Codecademy)
  • Computer Science (Stanford University on Coursera)
  • Douglas Crockford's lecture series on JavaScript

Week 1 Static Blog
  • Implement Jekyll and Markdown to quickly develop and release a web page
  • Use Bootstrap and CSS to make it beautiful
  • Utilise Git to manage your team's code

Week 2 Guardian news feed
  • Learn AJAX and pull data from an external API
  • Manipulate and alter JSON data to your needs
  • Integrate JavaScript and insert data into your page as it arrives
  • Use DevTools to debug and optimise code

Week 3 Your first server
  • Build a server using Node.js that responds with static pages and files
  • Make it modular: split your code into nimble packages and use libraries
  • Make it lightning fast: build asynchronous functionality

Week 4 Twitter gallery
  • Create a server that automatically archives Twitter images with a certain tag
  • Manage data from the Twitter API, and create a well-designed page to display images
  • Back up your Twitter data with a noSQL database such as MongoDB
  • Push your server to a cloud host such as Heroku

Week 5 Dynamic data visualisation
  • Introduction to the extraordinary D3 library
  • Learn how to dynamically render a map using data points
  • Load and toggle between data sets instantaneously
  • Get a taste for statistics and how powerful visualisation can be

Week 6 Blogging platform
  • Build a JSON API with a Hapi server using RESTful routes
  • Use a pooled database connection to save and retrieve entries and comments quickly
  • Easily render different views with dynamic data with Jade or Swig
  • Validate user input data with Joi, to keep your server happy
  • Integrate Rich-text editing with a plug-in such as TinyMCE

Week 7 Your first content management system
  • Adapt your blog platform into a full content management system
  • Allow multiple users to write rich content posts and publish them
  • Create a registration system as well as authentication with other options such as Facebook using Bell
  • Allow all users to leave comments on posts to get the conversation going
  • Enable caching to serve web browsers faster and to lighten the load on your database
  • Test your Hapi server with Lab, to make sure it is behaving correctly

Week 8 Make it Angular
  • Explore the basics of Angular.js by creating a basic store in Codeschool
  • Recreate the todoMVC project with a Hapi server and Angular.js
  • Replace your blogging platform front-end with Angular.js
